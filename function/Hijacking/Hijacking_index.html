劫持分析

本功能目前支持 DNS 劫持分析，DNS 劫持又称域名劫持。当 APP 发起 http 请求时，ISP 对相应的域名进行 DNS 解析，定向到错误的 IP 地址。 OneAPM 提供劫持白名单的方式来进行 DNS 监控，一旦出现白名单之外的 IP 地址即为 DNS 劫持，功能包括：

  • DNS 劫持趋势变化图
  • 多维度指标分析
  • DNS 劫持列表
  • 劫持白名单设置

DNS 劫持趋势变化图

首先提供劫持数和劫持率的时间趋势图。

  • 劫持数：被劫持的数量在时间段内的分布
  • 劫持率：被劫持的数量比照总的请求数量在时间段内的分布

多维度指标分析

展示 APP 版本，设备，操作系统，地域，网络类型维度的劫持数，支持选取任一维度或多个维度的联动组合。

DNS 劫持列表

统计各域名下的劫持数，展开域名显示各劫持 IP 的劫持数，列表内容支持选定某域名或ip后的整个页面的过滤。

劫持白名单设置

DNS 劫持采用白名单的方式，所有没有在名单中出现的 DNS 解析后的 IP 识别为劫持. 是否开启DNS劫持，默认是否，开启后计算劫持数，否则都为 0

白名单编辑：通过添加IP范围项或者特定的ip作为白名单列表，示例:
192.168.1.1 /192.168.1.24 包含192.168.1的网段的所有ip
202.21.21.1 表示这个特定的ip
