用户

我们从用户视角出发，实时追踪用户的每一次会话，并将用户会话过程中的所有网络请求、事件数、崩溃等性能数据与用户行为和用户信息绑定，从而监测每一次用户会话的真实体验。
该模块支持以下功能:

  • 用户会话总览
  • 多维度指标分析
  • 用户会话列表

用户会话总览

首先展示用户会话历史趋势图，并对用户访问触发到的网络请求、用户事件、崩溃、网络错误等关键性能指标数据进行预览，其中：

  • 会话数：选定时间范围内用户的会话次数，打开 APP 或者 30 分钟以上无动作再次操作 APP 时，会话次数增加一次；
  • 崩溃数：指选定时间范围内发生 APP 崩溃闪退的次数；
  • 用户请求数：指选定时间范围内用户的网络请求次数；
  • 网络请求错误数：指选定时间范围内发生网络请求错误的请求次数；
  • 事件数：指选定时间范围用户与 APP 交互的事件数；
  • 缓慢事件数：指响应时间超过 300 ms 的用户事件数；
  • 自身服务平均时间：指选定时间范围内属于自身网络服务的网络请求的平均响应时间；
  • 第三方服务平均响应时间：指选定时间范围内属于第三方网络服务的网络请求的平均响应时间；

多维度指标分析

根据用户会话在应用版本、操作系统、设备、地域的分布概况，支持从任意指标维度出发进行多维度组合分析。

用户会话列表

用户访问列表实时展示用户每一次访问下的用户信息、开始时间、应用版本、操作系统版本、设备、地域、缓慢事件数、网络请求错误数、崩溃数。

点击任意一条用户会话，即可跳转至用户访问轨迹页面，查看用户在该次会话访问中的基础信息：

  • 用户信息：默认为 DeviceID，用户可通过 Mi 接口配置自定义的用户信息；
  • 开始时间：用户一次会话的开始时间；
  • 持续时间：用户从会话开始到用户关闭 APP／APP 发生崩溃 ／用户访问超时（超过 30 分钟以上无动作）的整体耗时；
  • 版本号：应用版本号；
  • 是否有 Crash :该次会话中用户是否发生崩溃；
  • 用户满意度：根据 UEI 值进行评估，（ 0-60 分）显示为不满意，（ 60-80 分）显示为一般，（ 80-100 分）显示为满意；
  • 联网方式：用户联网方式，例如中国移动 3G／4G 等；
  • 地理位置：用户设备地理位置；
  • 用户动作总数：一次会话过程中用户与 APP 所有的用户动作数；
  • 缓慢动作数：动作执行响应时间超过 300 ms （可调）的动作总数；
  • 设备：用户设备类型及操作系统；

  • 用户轨迹：用户会话过程中随时间变化的动作轨迹，其中正常动作采用聚合方式展示，缓慢动作单独呈现，采用红色标注执行时间；
    点击任一缓慢动作，可跳转至动作详情页，可查看该慢动作耗时详细信息，包括总耗时、网络请求数、网络慢请求次数、是否有 Error （网络）,可用运行内存、数据发送量、数据接收量。另外，绘制该动作触发的页面交互和网络请求资源时序图，针对具体的页面交互和网络请求，均可查看具体耗时情况。

  • HTTP 响应时间：从发送网络请求到接收完服务器的响应数据耗时；

  • 首字节时间：从发送 HTTP 请求到接收服务器返回的首个字节的传输时间；

  • HTTP方法：POST/GET ；

  • 错误：发生 HTTP 错误或者网络错误的错误类型；

  • 发送数据：网络请求发送的数据量大小；

  • 接收收据：网络请求接收的数据量大小；

对于任一网络请求，均可点击请求详情，查看具体的网络请求信息。
