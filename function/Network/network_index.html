网络请求

网络请求功能监测分析 APP 交互过程中的网络请求相关性能指标，帮助我们定位网络性能问题。该界面包含以下内容：

  • 网络请求变化趋势图
  • 指标对比分析
  • 多维度指标分析
  • 用户请求追踪列表

网络请求时间趋势图

展示选定时间范围内网络请求的响应时间、传输量变化趋势图
响应时间：指平均响应时间，选定时间范围内网络请求响应总时间与请求次数之比。
传输量：指平均传输量，选定时间范围内用户所有网络请求的传输数据量与请求次数之比

指标对比分析

展示缓慢请求与正常请求对比柱状图
缓慢请求：默认响应时间阈值大于或等于 300 ms （可设置）的网络请求为慢请求。
正常请求：默认响应时间阈值小于 300 ms（可设置） 的网络请求为正常请求。

多维度指标分析

网络请求／慢请求／错误请求在应用版本、地域、联网方式、系统版本、设备等维度的饼状图分布，点击某项指标可进行维度筛选
错误请求：网络请求状态码大于等于 400 的 HTTP 错误和发生 tcp 连接问题、dns 问题等网络错误的网络请求总和。
比如当您想要查看某一应用版本下，联网方式为 WI-FI 的缓慢请求，可以在维度分析中选择缓慢请求维度 ，点击饼状图中您想要的应用版本分区和联网方式分区，即可查看：

用户请求追踪列表

展示用户在选定时间范围内所有的网络请求次数、响应时间、传输量、缓慢请求次数、错误率、首包时间。 响应时间：指平均响应时间，等于选定时间范围内所在域名网络请求总次数与网络请求总响应时间之比；
错误率：指发生 HTTP 错误数和网络故障的请求数与总请求数之比；
首包时间：指从发送 HTTP 请求结束开始，到收到服务器返回的第一个数据包的消耗时间；
点击用户请求列表中任一网络请求，查看单条网络请求数据性能状况。 点击缓慢请求次数，可跳转至慢请求详情页面，展示此次缓慢请求时的响应时间、响应码、错误码等详情信息，其中用户还可通过自定义设置是否展示请求参数、请求头、返回头等具体请求信息。
